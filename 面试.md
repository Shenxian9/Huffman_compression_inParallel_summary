# 项目定位
这是一个 **Huffman 压缩算法并行化对比项目**，核心目标是在统一压缩格式下实现并对比：串行版本、OpenMP 版本、CUDA 版本，并通过 controller 统一调度运行。

- 核心目标：比较不同并行模型在同一任务上的性能收益与工程代价
- 主要技术栈：C++、OpenMP、CUDA、命令行交互与文件 I/O
- 最易被追问方向：并行策略、块化设计、哈夫曼树实现、GPU 内核瓶颈、跨平台工程化

# 2分钟项目介绍稿
各位老师好，我的项目是一个 Huffman 压缩并行化对比实现。在统一压缩格式下，我实现了串行、OpenMP 和 CUDA 三个版本，并设计了统一 controller 进行调度运行。公共模块负责字符统计、树节点和哈夫曼树构建，三套实现共享协议，便于互相解压验证。

算法流程是读文件、统计频次、构树、生成编码表、按块打包写入；解压则逆过程恢复。块化设计是关键，它让每块可独立解码，天然支持并行。

OpenMP 版主要并行了频次统计、压缩分段和逐块解压；CUDA 版采用 CPU 构树 + GPU 统计/编码/解码的混合方案。项目中我也识别了性能瓶颈，比如 OpenMP 的归并热点和 CUDA 编码内核线程利用率偏低。

后续优化上，我会优先做两件事：一是 OpenMP 去 critical 的局部桶归并；二是 CUDA 编码改成块内多线程协作打包。这个项目对我最大的价值是把数据结构、并发、GPU 和工程化整合到一个可比较、可解释、可扩展的完整实践里。

# 30秒项目介绍稿
我做的是 Huffman 压缩的三版本并行化项目：串行、OpenMP、CUDA，统一协议、统一入口。亮点是块化设计让解压天然并行，OpenMP 和 CUDA 都围绕块粒度做加速。项目同时关注性能和工程可用性，比如跨平台 controller 与错误前置提示。下一步会重点优化 OpenMP 归并和 CUDA 编码内核并行度。
