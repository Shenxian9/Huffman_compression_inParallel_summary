这个项目的目标是把 Huffman 压缩/解压从串行扩展到 OpenMP 和 CUDA 两种并行实现，在 1MB 到 1GB 数据集上对比时间、加速比和压缩比，回答“在什么场景下 CPU 并行更合适、什么场景下 GPU 更有优势”。

## Q2
### 问题
为什么选 Huffman 做并行优化？
### 考察意图
看选题合理性。
### 回答要点
- Huffman 有明确流程：统计→构树→编码/解码
- 部分阶段并行性强（统计、块处理）
- 适合做串行/并行对照
### 示范回答
Huffman 算法流程清晰，既有容易并行的部分（频率统计、块级处理），也有相对串行的部分（构树、前缀匹配），非常适合做并行优化的教学型和研究型对比。
### 危险回答
“因为 Huffman 最简单。”
### 可扩展追问
哪一步最难并行化？

## Q3
### 问题
串行、OpenMP、CUDA 三个版本在流程上有哪些共性？
### 考察意图
看你是否真的做了“同任务对比”。
### 回答要点
- 都有压缩/解压闭环
- 都基于同一编码逻辑和块化格式
- 差别主要在并行执行方式
### 示范回答
三版共性是同一 Huffman 逻辑和同一块化数据组织，保证可比性。差异只在执行模型：串行单线程、OpenMP 多核线程并行、CUDA 用 GPU kernel 做高并发处理。
### 危险回答
“其实三个版本不太一样，无法比较。”
### 可扩展追问
你如何保证比较公平？

## Q4
### 问题
OpenMP 版本你并行化了哪些环节？
### 考察意图
检查你是否掌握代码层细节。
### 回答要点
- 并行频率统计
- 并行块压缩

### 示范回答
OpenMP 主要并行了三个热点：频率统计、分块压缩和逐块解压。频率统计采用线程私有桶再归并，压缩和解压都按块分配给线程处理。

## Q7
### 问题

### 考察意图
落实到实现细节。
### 回答要点

## 第2轮：实现细节
老师：OpenMP 和 CUDA 具体各优化了什么？
你：OpenMP 主要并行统计和块级编解码；CUDA 主要并行频率统计和块处理，shared memory 用于块内临时打包，减少全局访存。

## 第3轮：实验与结论
老师：那你最后怎么判断哪种方案更好？

# 2分钟项目介绍稿（简历版）
各位老师好，我的项目是“基于多核 CPU 与 CUDA 的并行 Huffman 压缩优化”。项目周期是 2024 年 12 月到 2025 年 1 月，核心目标是在统一任务下比较串行、OpenMP 与 CUDA 三种实现的性能与适用场景。

在实现上，我完成了压缩/解压全流程三版本：串行版作为基线；OpenMP 版重点做并行频率统计和块级并行编解码；CUDA 版在 GPU 上做高并发频率统计与块级处理，并在编码阶段使用 shared memory 缓冲块内数据，减少全局内存访问开销。

为了让并行更可落地，我采用了分块压缩策略，使块之间相对独立，便于线程和 GPU block 调度。评估方面，我用 1MB 到 1GB 数据集，从压缩时间、解压时间、加速比、压缩比四个指标进行对比，重点不是追求单一“最快”，而是找出不同规模下 OpenMP 和 CUDA 的收益边界。

这个项目让我把数据结构、并行编程和性能评测连在一起，也让我理解了并行优化的本质是“正确性、开销、硬件条件”三者平衡。
